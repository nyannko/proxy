V2

1.预编译安装

(不用使用sudo且不用自己配置go环境的，适合das5的)

1.在https://github.com/v2ray/v2ray-core/releases下载最新的release

2.unzip，注意后面要写上文件夹的名字

unzip v2ray-linux-64.zip -d v2ray-linux-64

3.设置配置文件 mkdir selfconfig; cd selfconfig

server:

vi server.json

{
  "inbounds": [{
    "port": 20000,
    "protocol": "vmess",
    "settings": {
      "clients": [
        {
          "id": "c04abb12-ff01-11e8-8eb2-f2801f1b9fd1",
          "level": 1,
          "alterId": 64
        }
      ]
    }
  }],
  "outbounds": [{
    "protocol": "freedom",
    "settings": {}
  },{
    "protocol": "blackhole",
    "settings": {},
    "tag": "blocked"
  }],
  "routing": {
    "rules": [
      {
        "type": "field",
        "ip": ["geoip:private"],
        "outboundTag": "blocked"
      }
    ]
  }
}

client: 

vi client.json

{
  "inbounds": [{
    "port": 1080,  // SOCKS 代理端口，在浏览器中需配置代理并指向这个端口
    "listen": "127.0.0.1",
    "protocol": "socks",
    "settings": {
      "udp": true
    }
  }],
  "outbounds": [{
    "protocol": "vmess",
    "settings": {
      "vnext": [{
        "address": "你的serverip", // 服务器地址，请修改为你自己的服务器 ip 或域名
        "port": 20000,  // 服务器端口
        "users": [{ "id": "c04abb12-ff01-11e8-8eb2-f2801f1b9fd1" }]
      }]
    }
  },{
    "protocol": "freedom",
    "tag": "direct",
    "settings": {}
  }],
  "routing": {
    "strategy": "rules",
    "settings": {
      "domainStrategy": "IPOnDemand",
      "rules": [{
        "type": "field",
        "ip": ["geoip:private"],
        "outboundTag": "direct"
      }]
    }
  }
}

4.启动client和server，并测试

cd ..
./v2ray -config selfconfig/client.json
./v2ray -config selfconfig/server.json
curl -I --proxy socks5h://127.0.0.1:7000 www.google.com

3.手动编译v2ray

起因是因为测出来的的vmess的各种加密方式的速度基本上一样，和理论不是很符合，就想修改源代码debug看哪里错了

  • 安装go，版本12以上（支持tls）

  • 编译安装，这边有个flag很坑，看了源代码才说不需要加，否则会出现包import问题

cd $(go env GOPATH)/src/v2ray.com/core/main env
CGO_ENABLED=0
go build -o $HOME/v2raytest/v2ray -ldflags "-s -w"
cd $(go env GOPATH)/src/v2ray.com/core/infra/control/main
env CGO_ENABLED=0
go build -o $HOME/v2raytest/v2ctl -ldflags "-s -w"
  • 配置文件

想查看vmess协议是否使用了某种加密方式，但是不知道是不是配置文件写的不对的问题，一直出现错误（但是流量还是可以正常转发），在config入口设置了写入日志的消息也并没有被打印。

有两种可能：

  1. 配置文件写错
  2. 消息打印设置错了位置（✔️)
